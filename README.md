# Dotfiler

A clean, organized dotfiles management system following zero-magic principles. Portable, predictable, and works on any Unix-like machine.

## Features

- **Auto-Detection**: Self-locating scripts work from any directory location
- **Organized Structure**: Files grouped by purpose, not file type
- **One-Command Setup**: `./install.sh` handles everything
- **Modular Loading**: Clear, predictable configuration loading
- **Dependency Management**: Centralized package management with Homebrew
- **Development Ready**: Includes shell functions, scripts, and development tools

## Quick Start

```bash
# Place dotfiler anywhere you like
git clone <this-repo> ~/dotfiler
cd ~/dotfiler

# Initialize (generates platform-specific mappings.yaml)
./init.sh

# Run the installer
./install.sh

# Start using (or reload current shell)
source ~/.zshrc
```

### Bringing Existing Dotfiles

Already have dotfiles? Bring them under management:

```bash
# Interactive mode
./bring.sh
# Wizard will prompt for file path, public/private, and confirmation

# Command-line mode (automation-friendly)
./bring.sh ~/.vimrc --public --yes
./bring.sh ~/.aws/credentials ~/.aws/config --private --yes

# Options:
#   --private    Mark as private (default)
#   --public     Mark as public
#   --yes, -y    Auto-confirm all prompts
#   --help, -h   Show usage examples
```

**What it does:**
1. Creates `.backup` of original file
2. Moves file to dotfiler (public or private)
3. Creates symlink at original location
4. Updates mappings.yaml automatically

## How It Works

1. **init.sh** detects your platform (macOS/Linux) and generates platform-specific `mappings.yaml`
2. **mappings.yaml** (generated) defines which files get symlinked and where - clean, no platform clutter
3. **install.sh** reads mappings.yaml and creates symlinks dynamically
4. **~/.zshrc** sources `shell/load.sh` from the detected location
5. **shell/load.sh** systematically loads all configs, functions, scripts, and utils
6. **bin/** directory is automatically added to PATH
7. **Everything uses relative paths** - works from any location

## Customization

### Adding New Functionality
- **Shell functions**: Add to appropriate file in `shell/functions/`
- **Scripts**: Place in `shell/scripts/` directory
- **Tools**: Add executables to `bin/` directory
- **Packages**: Update `packages/Brewfile`

### Customizing File Mappings

All dotfile symlinks are configured in **`mappings.yaml`** (generated by `init.sh`). This file is clean and straightforward - no platform clutter!

**Quick start:**
```bash
# Edit mappings.yaml to add/remove/modify symlinks
vim mappings.yaml

# Run install to apply changes
./install.sh
```

**Example - Your mappings.yaml looks like this (clean!):**
```yaml
# Platform: macos
public:
  - source: zshrc
    target: ~/.zshrc
  - source: vimrc
    target: ~/.vimrc

private:
  - source: claude-desktop/config.json
    target: ~/Library/Application Support/Claude/claude_desktop_config.json
    # ^^ Only the macOS path, no Linux path clutter!
```

**Adding a new dotfile:**
```yaml
public:
  - source: vimrc          # File in dotfiles/
    target: ~/.vimrc       # Where to link it
```

**Regenerating for different platform:**
```bash
# If you move to different OS or want to regenerate
./init.sh  # Detects new platform, regenerates mappings.yaml
```

### Bringing Existing Files Under Management

Already have dotfiles scattered around your system? Use `bring.sh` to adopt them:

**Interactive mode:**
```bash
./bring.sh

Enter path to bring: ~/.config/claude/claude_desktop_config.json
Is this PUBLIC? [y/N]:  # Press Enter (defaults to private)
Accept suggested destination? [Y/n]: y
Proceed? [y/N]: y

[✓] Created backup: ~/.config/claude/claude_desktop_config.json.backup
[✓] Moved to: dotfiles-private/claude-desktop/config.json
[✓] Created symlink
[✓] Updated mappings.yaml
```

**Command-line mode (automation-friendly):**
```bash
# Single file
./bring.sh ~/.vimrc --public --yes

# Multiple files at once
./bring.sh ~/.aws/credentials ~/.aws/config --private --yes

# With spaces in path (use quotes)
./bring.sh "~/Library/Application Support/Claude/config.json" --private --yes

# View all options
./bring.sh --help
```

**Options:**
- `--private` - Mark as private dotfile (default)
- `--public` - Mark as public dotfile
- `--yes, -y` - Auto-confirm all prompts (no interaction)
- `--help, -h` - Show detailed usage and examples

**What it does:**
1. **Backs up** original file (`.backup` - excluded from git)
2. **Moves** file to dotfiler (private or public)
3. **Creates symlink** from original location
4. **Updates** mappings.yaml automatically
5. **No install.sh needed** - works immediately!

**Features:**
- Defaults to private (secure by default)
- Smart destination suggestions based on file path
- Always creates backup before moving
- Supports multiple files in one command
- Handles paths with spaces correctly

### Managing Private/Sensitive Configs

For credentials, API keys, and other sensitive information:

1. **Create private directory structure**:
   ```bash
   mkdir -p dotfiles-private/claude-desktop
   ```

2. **Copy from example template**:
   ```bash
   cp examples/claude-desktop-config.example.json \
      dotfiles-private/claude-desktop/config.json
   ```

3. **Edit with your real credentials**:
   ```bash
   vim dotfiles-private/claude-desktop/config.json
   ```

4. **Verify/add mapping in `mappings.yaml`** (common configs already included):
   ```yaml
   private:
     - source: claude-desktop/config.json
       target: ~/Library/Application Support/Claude/claude_desktop_config.json
   ```

5. **Run install to create symlink**:
   ```bash
   ./install.sh
   ```

**Security**: The entire `dotfiles-private/` directory is excluded from git, so your credentials will never be committed.

## Troubleshooting

### When to Run install.sh

Run `./install.sh` if:
- ✅ Initial setup on a new machine
- ✅ Symlinks are broken or pointing to wrong locations
- ✅ After moving the dotfiler directory
- ✅ Missing dotfiles in home directory

**To check if symlinks are correct:**
```bash
ls -la ~/.zshrc    # Should point to /path/to/dotfiler/dotfiles/zshrc
ls -la ~/.gitconfig # Should point to /path/to/dotfiler/dotfiles/gitconfig
```

**Don't run install.sh if:**
- ❌ Just editing config files (use `reload` instead)
- ❌ Adding new functions or aliases (use `reload` instead)
- ❌ Symlinks are already correct

### Changes Not Taking Effect?

1. **Check symlinks first:**
   ```bash
   ls -la ~/.zshrc
   ```
   If pointing to old location (e.g., `~/.shell/`), run `./install.sh`

2. **For correct symlinks, just reload:**
   ```bash
   reload  # or: source ~/.zshrc
   ```

3. **Verify the change worked:**
   ```bash
   alias c        # Check alias definition
   echo $EDITOR   # Check environment variable
   ```
